# 项目重构说明

## 一、重构目标

本次重构以毕设题目“基于Python的智能垃圾分类与回收处理系统”为核心，围绕以下目标展开：

1. 支持图片中多个垃圾目标同时识别；
2. 支持视频与摄像头连续识别；
3. 在视频中兼顾识别准确性、流畅度与边界框跟随度；
4. 识别后联动垃圾分类知识、回收建议与回收点检索；
5. 针对 GTX 1650 显卡给出可落地的训练方案。

## 二、关键调整

### 1. 类别体系调整
- 原 `cardboard` 已合并到 `paper`
- 最终保留 9 类

### 2. 数据层调整
- 停止使用整图伪标注
- 改为真实框 YOLO 标签整理流程
- 新增标签校验和数据统计脚本

### 3. 模型层调整
- 继续以 YOLOv8 为主干
- 提供 GTX1650 对应的 `YOLOv8n` 和 `YOLOv8s` 两种两阶段训练配置
- 视频侧加入轻量 IoU 跟踪、框平滑和类别投票

### 4. 系统层调整
- Node.js 负责统一 API 网关、历史记录与回收点检索
- Python 服务负责模型推理
- 前端统一支持图片、视频、摄像头、知识库与历史记录展示

## 三、为什么这样改

原因在于你当前项目的真实目标已经不再是单目标图像分类，而是：

- 多目标垃圾检测
- 视频连续识别
- 边界框稳定跟随
- 回收处理闭环

因此项目必须从“演示型识别页面”升级为“检测系统”。
