# 完整项目构建顺序（适配 GTX 1650）

## 阶段1：整理目标与类别
1. 确定题目定位为多目标检测系统；
2. 将 cardboard 合并到 paper；
3. 固定为 9 类，不再频繁改类别。

## 阶段2：构建真实框数据集
1. 采用 LabelImg 标注；
2. 优先保证验证集全是真实框；
3. 先做核心高质量数据，再逐步扩充；
4. 运行 `validate_labels.py` 检查标签有效性；
5. 运行 `dataset_stats.py` 生成数据分布统计。

## 阶段3：整理标准 YOLO 数据集
1. 运行 `prepare_yolo_dataset.py`；
2. 生成 `dataset/images` 与 `dataset/labels` 目录；
3. 配置 `data.yaml`。

## 阶段4：先做轻量训练验证
1. 使用 `YOLOv8n`；
2. `imgsz=640`；
3. `batch=4~8`；
4. 先用小样本试训，确认标签和脚本无误。

## 阶段5：正式训练
1. 运行 `python train_pipeline.py --profile gtx1650_n`；
2. 观察 loss、mAP、样例图；
3. 若效果稳定，再尝试 `gtx1650_s` 作为增强版。

## 阶段6：完成图片识别链路
1. Python 服务加载 `best.pt`；
2. Node 网关转发图片识别请求；
3. 前端显示多目标框、类别、置信度、建议。

## 阶段7：完成视频识别
1. 采用间隔帧检测；
2. 对中间结果做 IoU 时序跟踪；
3. 对 bbox 做平滑；
4. 对类别做投票；
5. 输出结果视频。

## 阶段8：完成摄像头识别
1. 浏览器采集视频帧；
2. 调用 `/api/predict/frame`；
3. 前端 canvas 绘制实时框；
4. 加入 tracker reset。

## 阶段9：完成闭环功能
1. 历史记录保存；
2. 垃圾知识库；
3. 回收建议展示；
4. 回收点检索。

3. 图片识别效果图；

5. 模型选择与 GTX1650 约束说明。
